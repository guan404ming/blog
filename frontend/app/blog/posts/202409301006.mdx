## 📌 [unclecode/crawl4ai](https://github.com/unclecode/crawl4ai)
<Callout>
    Description: 🔥🕷️ Crawl4AI: Open-source LLM Friendly Web Crawler & Scrapper\
    🌐 Python｜⭐️ 11,367 | 7009 stars this week
</Callout>
    #### 簡介

Crawl4AI 是一個非同步網頁爬取和資料擷取工具，專為大型語言模型 (LLM) 和 AI 應用程式設計，讓您輕鬆進行網頁爬取和資料擷取。

- 完全免費且開源。
- 效能極佳，超越許多付費服務。
- 支援 LLM 友善的輸出格式，例如 JSON、清除 HTML、markdown。
- 支援同時爬取多個網址。
- 擷取並返回所有媒體標籤 (影象、音訊和影片)。
- 擷取所有外部和內部連結。
- 擷取頁面的金屬資料。
- 可自訂鉤子，在爬取之前進行驗證、標頭和頁面修改。
- 可自訂使用者代理。
- 可擷取頁面的螢幕截圖。
- 可在爬取之前執行多個自訂 JavaScript。
- 使用 JsonCssExtractionStrategy 生成結構化輸出，無需 LLM。
- 多種切分策略，例如主題、正規表示式、句子等等。
- 進階擷取策略，例如餘弦叢集、LLM 等等。
- 支援 CSS 選擇器，實現精準資料擷取。
- 可以傳遞指令/關鍵字以精煉擷取結果。
- 支援代理，提高隱私和訪問許可權。
- 具有會話管理功能，方便處理複雜的多頁面爬取場景。
- 非同步架構，提升效能和可擴充套件性。

#### 主要功能

Crawl4AI 提供以下主要功能：

- **快速非同步網頁爬取：** 使用 Playwright 進行非同步爬取，快速獲取網頁內容。
- **資料擷取：** 提供多種擷取策略，包括 CSS 選擇器、LLM 和正規表示式，支援精準資料擷取。
- **結構化輸出：** 支援 JSON、清除 HTML 和 markdown 等輸出格式，方便 LLM 處理。
- **會話管理：** 支援多頁面爬取，方便處理動態內容。
- **自訂鉤子：** 支援自訂鉤子，方便在爬取過程中進行自定義操作，例如驗證、標頭修改等。
- **代理支援：** 支援代理，提高隱私和訪問許可權。
- **Docker 支援：** 提供 Docker 映象，方便在容器化環境中使用。

#### 如何使用

- **安裝 Crawl4AI：** 使用 pip 安裝 Crawl4AI，並根據需要選擇安裝選項，例如同步版本或開發版本。
- **基本用法：** 使用 `AsyncWebCrawler` 類別進行爬取，並使用 `arun` 方法執行爬取任務。
- **進階用法：** 可使用 CSS 選擇器、JavaScript 執行、代理設定、自訂擷取策略等功能，進行更複雜的爬取任務。
- **範例：** 參考 README.md 中的範例，學習如何使用 Crawl4AI 進行各種爬取任務。
- **檔案：** 訪問 Crawl4AI 的檔案網站，瞭解更多資訊和詳細說明。

## 📌 [tw93/Pake](https://github.com/tw93/Pake)
<Callout>
    Description: 🤱🏻 Turn any webpage into a desktop app with Rust. 🤱🏻 利用 Rust 轻松构建轻量级多端桌面应用\

    #### 簡介

Pake 是一款用 Rust 開發的工具，可以輕鬆地將任何網頁轉換為桌面應用程式。它支援 Mac、Windows 和 Linux，並且比基於 JS 的框架更輕量級且更快。Pake 提供了預設的套件功能，例如捷徑傳輸、沉浸式視窗和簡潔的客製化選項。Pake 只是個簡單的工具，可以取代舊的套件方法，並使用 Tauri 進行包裝（儘管 PWA 已經足夠）。

#### 主要功能

* Pake 的體積比 Electron 套件小 20 倍（約 5M）。
* Pake 使用 Rust Tauri，比基於 JS 的框架更輕量級且更快。
* 內建套件功能：捷徑傳輸、沉浸式視窗和簡潔的客製化選項。
* Pake 只是個簡單的工具，可以取代舊的套件方法，並使用 Tauri 進行包裝（儘管 PWA 已經足夠）。

#### 如何使用

* **入門：** 可以試用 Pake 的預設套件，瞭解其功能，也可以嘗試使用 GitHub Actions 包裝自己的應用程式。若有疑問，請在討論區尋求幫助。
* **開發者：**  "指令列包裝" 支援 macOS。Windows 和 Linux 使用者可能需要調整設定。開始前請先設定您的環境。
* **開發者：**  熟悉前端開發和 Rust 的使用者，可以嘗試使用客製化開發功能來進一步自訂應用程式的功能。

* **使用指令列包裝：**  Pake 提供指令列工具，讓套件客製化流程更快捷、更方便。請參閱檔案瞭解更多資訊。
    * 使用 npm 安裝： `npm install -g pake-cli`
    * 使用指令： `pake url [OPTIONS]...`
    * 使用範例：`pake https://weekly.tw93.fun --name Weekly --hide-title-bar`
    * 也可以使用 GitHub Actions 線上編譯套件。請參閱教學以瞭解更多資訊。

* **開發：**  開始前請先設定好環境。確保您的電腦上已安裝 Rust >= 1.63 和 Node >= 16（例如 16.18.1）。安裝指南請參閱 Tauri 檔案。
    * 安裝依賴： `npm i`
    * 本地開發： `npm run dev`
    * 包裝應用程式： `npm run build`

* **進階使用：**  在開發 Pake 前可以參考程式碼結構，這將對您的開發工作有很大幫助。
    * 在 src-tauri 目錄下的 pake.json 檔案中修改 url 和 productName 欄位，同時需要修改 tauri.config.json 檔案中的 "domain" 欄位，以及 tauri.xxx.conf.json 檔案中的 icon 和 identifier 欄位。您可以從 icons 目錄中選擇圖示，或從 macOSicons 下載圖示以符合產品需求。
    * 您可以修改 pake.json 檔案中的 windows 屬性，更改 width、height、fullscreen（或不）、resizable（或不）的值。要適應 Mac 上的沉浸式標題，請將 hideTitleBar 設定為 true，並尋找 Header 元素，然後新增 padding-top 屬性。
    * 有關進階用法，例如樣式重寫、廣告移除、JS 注入、容器訊息通訊和使用者自訂的快捷鍵，請參閱 Pake 的進階使用。

## 📌 [exo-explore/exo](https://github.com/exo-explore/exo)
<Callout>
    Description: Run your own AI cluster at home with everyday devices 📱💻 🖥️⌚\
    🌐 Python｜⭐️ 9,583 | 2384 stars this week
</Callout>
    #### 簡介

exo 是一個開源軟體，允許使用者使用現有的裝置建立自己的 AI 叢集，例如 iPhone、iPad、Android、Mac、Linux 等。exo 不需要昂貴的 NVIDIA GPU，可以將所有裝置整合為一個強大的 GPU。exo 還在開發中，可能會遇到 bug，使用者可以回報問題以幫助開發團隊快速解決。

#### 主要功能

* **支援多種模型**: exo 支援 LLaMA (MLX 和 tinygrad)、Mistral、LlaVA、Qwen 和 Deepseek 等不同模型。
* **動態模型分割**: exo 會根據目前的網路拓撲結構和可用裝置資源，最佳化分割模型。
* **自動裝置偵測**: exo 會自動偵測其他裝置，無需手動配置。
* **ChatGPT 相容的 API**: exo 提供 ChatGPT 相容的 API，用於執行模型。
* **裝置平等**: 與其他分散式推論框架不同，exo 不使用主從架構。相反，exo 裝置會使用 p2p 連線。只要裝置連線到網路中的任何位置，都可以用於執行模型。

#### 如何使用

* **從原始碼安裝**: 使用 git clone 命令克隆 exo 儲存庫，然後使用 pip install 命令安裝 exo。
* **硬體需求**: 確保所有裝置的總記憶體足以容納整個模型。
* **效能**: 可以在 Apple Silicon Macs 上升級至最新版的 MacOS 15，並執行 ./configure_mlx.sh 命令來最佳化 GPU 記憶體配置，以提升效能。
* **範例用法**: 在多個 MacOS 裝置上使用 exo，只需在每個裝置上執行 exo 命令即可。exo 會自動偵測其他裝置，並在 http://localhost:8000 上啟動 ChatGPT 類似的 WebUI。
* **開發者 API**: exo 還會在 http://localhost:8000/v1/chat/completions 上啟動 ChatGPT 相容的 API 端點。使用者可以使用 curl 命令傳送請求，例如執行 Llama 3.2 3B 或 Llama 3.1 405B 等模型。
* **異構裝置**: exo 也支援異構裝置，例如 macOS 和 Linux。使用者可以在 macOS 裝置上執行 exo --inference-engine tinygrad 命令，而在 Linux 裝置上執行 exo 命令。
* **tinygrad 環境變數**: tinygrad 支援一些環境變數，例如設定 tinygrad 使用 CPU。
